# Локализация интерфейса (RU / EN)

Документ фиксирует реализацию переключения языков в приложении: стек, файлы, правила выбора языка и как добавлять новые строки. См. также индекс **`[docs/README.md](./README.md)`** (другие разделы документации).

---

## Стек

| Пакет | Назначение |
|--------|------------|
| **`i18next`** | Загрузка ресурсов, интерполяция (`{{title}}`), множественное число по правилам языка. |
| **`react-i18next`** | Хук **`useTranslation`**, провайдер **`I18nextProvider`**. |

Зависимости указаны в корневом **`package.json`** (`i18next`, `react-i18next`).

---

## Файлы переводов

Строки разнесены по двум JSON-файлам (один язык — один файл):

| Файл | Язык |
|------|------|
| **`src/locales/en.json`** | Английский (fallback). |
| **`src/locales/ru.json`** | Русский. |

Структура ключей в обоих файлах зеркальная. В коде используются ключи с точками, соответствующие вложенности в JSON, например **`layout.nav.boards`**, **`kanban.column.deleteTitle`**.

Группы ключей (кратко):

- **`layout`** — бренд, навигация, кнопки входа/выхода, подписи языка в селекте.
- **`common`** — общие кнопки и подписи (отмена, удаление, загрузка и т.д.).
- **`settingsPage`**, **`boardsPage`**, **`boardPage`** — страницы.
- **`boardsSidebar`**, **`boardSettings`** — правый сайдбар и диалог настроек доски.
- **`authDialog`** — диалоги входа и регистрации.
- **`kanban`** — доска, колонки, формы, карточки задач (`board`, `column`, `columnForm`, `taskCard`, `taskForm`), панель **`aiBreakdown`** (разбор задачи через Worker; см. **[docs/VENICE-BREAKDOWN.md](./VENICE-BREAKDOWN.md)**).

## Инициализация и хранение языка

Файл **`src/lib/i18n.ts`**:

1. Читает сохранённый язык из **`localStorage`** под ключом **`preferred-locale`** (`ru` или `en`).
2. Если ключа нет — выбирает **`ru`**, если **`navigator.language`** начинается с `ru`, иначе **`en`**.
3. Подписывается на **`languageChanged`**: обновляет **`document.documentElement.lang`** и снова записывает значение в **`localStorage`**.

Экспорт:

- **`changeAppLocale(lang)`** — сменить язык из UI (`AppLocale`: `'ru' | 'en'`).
- **`LOCALE_STORAGE_KEY`**, **`SUPPORTED_LOCALES`**, тип **`AppLocale`**.

Подключение до React:

- **`src/main.tsx`** — в начале: **`import '@/lib/i18n'`**.
- **`src/app/providers.tsx`** — дерево обёрнуто в **`I18nextProvider`** с экземпляром **`i18n`** из **`@/lib/i18n`**.

## Переключатель в UI

В **`src/components/shared/AppLayout.tsx`** в шапке (слева от блока с email и кнопками авторизации): нативный **`<select>`** с значениями `en` / `ru`; подписи опций через **`layout.locale.en`** и **`layout.locale.ru`**. Смена вызывает **`changeAppLocale(...)`**.

## Компоненты, где используются переводы

Строки вынесены в JSON и подставляются через **`useTranslation()`** и **`t('ключ')`** (при необходимости второй аргумент: **`{ count }`**, **`{ title }`** и т.д.):

| Область | Файлы |
|---------|--------|
| Раскладка | **`AppLayout.tsx`** |
| Auth | **`AuthDialogs.tsx`** (включая разделитель «Or» и Google-кнопку) |
| Страницы | **`BoardsPage.tsx`**, **`BoardPage.tsx`**, **`SettingsPage.tsx`** |
| Доски / участники | **`BoardsRightSidebar.tsx`**, **`BoardSettingsDialog.tsx`** |
| Канбан | **`Board.tsx`** (в т.ч. **`BoardAiBreakdownPanel`**), **`Column.tsx`**, **`ColumnFormDialog.tsx`**, **`TaskCard.tsx`**, **`TaskCardFormDialog.tsx`** (в т.ч. **`CreateTaskCardDialog`**) |

Сообщения об ошибках с API (Supabase и т.п.) по возможности не переводятся — показывается текст от сервиса.

## Множественное число

Для ключей с **`count`** в JSON используются суффиксы **`_one`**, **`_few`**, **`_many`**, **`_other`** там, где правила русского языка этого требуют; для английского обычно достаточно **`_one`** и **`_other`**.

Примеры вызова в коде: **`t('boardsPage.count', { count })`**, **`t('boardPage.tasksCount', { count })`**, **`t('kanban.board.columnsCount', { count })`**.

## TypeScript и сборка

В **`tsconfig.app.json`** включено **`resolveJsonModule: true`** для импорта JSON в **`src/lib/i18n.ts`**.

## Как добавить новую строку

1. Добавить ключ и текст в **`src/locales/en.json`** и **`src/locales/ru.json`** (одинаковая вложенность).
2. В компоненте: **`const { t } = useTranslation()`**, затем **`t('group.key')`** или с параметрами.
3. Для длинных сообщений с подстановками использовать интерполяцию в JSON (**`{{title}}`**) и передать объект во второй аргумент **`t`**.
